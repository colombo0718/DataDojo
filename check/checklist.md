# DataDojo 專案檢核標準

> 供 Codex 定期巡檢，或每次重大功能上線前手動確認。
> 格式：`[ ]` 待確認，`[x]` 通過，`[!]` 有問題需處理。
>
> 檢核分三層：**目的層**（這個平台存在的理由）→ **教學層**（課堂實際能不能用）→ **功能層**（技術正不正確）。
> 功能層全過但目的層沒過，平台仍然是失敗的。

---

## 【目的層】平台定位自我檢驗

> 核心定位：填補「理論影片」與「Python 實操」之間的斷層。
> 這一層的問題沒有標準答案，需要人工主觀判斷。

- [ ] **理論到實作的橋**：學生能在看完教材理論後，立刻在 DD 上找到對應的操作，不需要額外說明
- [ ] **不需要程式碼**：完整走完一次 ML pipeline（載入→清理→訓練→評估），全程零行程式碼
- [ ] **比 Weka 更親民**：介面語言是中文、引導文字說明「為什麼」而不只是「做什麼」、不需要讀 manual 就能開始
- [ ] **調參數有即時回饋**：改變 k 值或樹深度後，學生能快速看到效果變化。（設計決策：滑桿只更新數值，需按「訓練」才重繪——這是刻意的，DT/RF auto-retrain 會造成明顯卡頓。評估重點是「調完參數到看到結果的摩擦感是否可接受」，不是滑桿是否即時觸發訓練）
- [ ] **老師看得見學生做了什麼**：紀錄 tab 有足夠的操作 log，能重建學生的操作順序

---

## 【教學層】課堂使用情境檢驗

> 想像一個技職高中的資料科學課，老師打開投影機，全班跟著操作。

- [ ] **第一次使用零門檻**：新使用者打開網頁，不看說明，60 秒內知道從哪裡開始、要做什麼
- [ ] **流程引導清楚**：六個 tab 的順序就是操作流程，學生不會不知道「下一步是什麼」
- [ ] **空狀態有提示**：在沒有訓練就切到成效 tab、沒有資料就切到訓練 tab 等情況下，顯示友善提示而不是空白頁面或錯誤
- [ ] **引導文字在關鍵處說明「為什麼」**：例如「為什麼要正規化」「k 值的意義是什麼」，不只說「請調整以下參數」
- [ ] **手機可用**：學生拿手機也能操作，portrait 模式下資料區與工具區分欄清楚
- [ ] **視覺化夠直觀**：一個對 ML 完全陌生的學生，看到決策邊界圖和混淆矩陣，有辦法猜出它在說什麼
- [ ] **不同程度都有東西看**：淺的學生看資料描述和分布圖就有收穫；深的學生能理解 F1 和 Elbow，同一個介面服務不同深度

## 【功能層】技術正確性驗收

### 資料載入與解析

- [ ] 七個內建資料集皆可正常載入：Iris、Palmer 企鵝、乳癌診斷、Titanic、心臟病診斷、Pima 糖尿病、商場顧客
- [ ] 無監督資料集（商場顧客）載入後，左側資料表無 target 欄
- [ ] CSV 上傳功能可正確解析數字標籤與文字標籤
- [ ] classMap 機制正常：Titanic / 心臟病 / 糖尿病的 0/1 數字對應中文類別名稱
- [ ] 切換資料集後，algorithm 重置為 k-NN，訓練結果清除

### 資料前處理 Tab

- [ ] 各欄位缺失值筆數正確偵測（有缺失值時橘色標示）
- [ ] 缺失值政策（補平均值 / 補中位數 / 刪除該筆）皆可正常執行
- [ ] Min-Max 縮放後欄位值域為 0~1
- [ ] Z-score 縮放後欄位均值≈0、標準差≈1
- [ ] 各欄位可個別設定不同縮放，互不干擾
- [ ] 套用後資料表顯示 3 位小數；重置後還原 1 位小數
- [ ] 前後直方圖切換預覽欄位後正確更新
- [ ] 套用前處理後，accuracy 和 predictor 自動清除（避免用舊模型）

### 訓練 Tab

- [ ] k-NN / Decision Tree / Random Forest 三種算法皆正常訓練並渲染決策邊界
- [ ] k-means 在無監督資料集下正常分群並渲染散點圖
- [ ] 各算法的參數滑桿（k / depth / 棵數 / 群數）正常運作
- [ ] X / Y 軸特徵選擇器切換後訓練結果清除

### 成效 Tab

- [ ] 二元分類：混淆矩陣 + precision / recall / F1 正確
- [ ] 多元分類：N×N 混淆矩陣正確，無二元指標行
- [ ] k-NN / DT / RF 各自的參數掃描曲線正確，紅點標記當前參數
- [ ] k-means：分群散點圖 + Elbow 曲線正確，紅點標記當前 k
- [ ] 未訓練即切到成效 tab 顯示提示，不崩潰
