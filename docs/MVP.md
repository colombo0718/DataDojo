# MVP.md — DataDojo 最小可行版本規劃

---

## MVP 要說的一個完整故事

> 選一個資料集 → 看看資料長什麼樣子 → 做正規化，看 before/after → 用 k-NN 訓練 → 拉動 k 值滑桿看決策邊界變形 → 知道 k 太小會過擬合

這個故事從頭走完，DD 的核心價值就被驗證了。

---

## UI 層面

### 左欄（處理對象）
- 內建資料集選擇器（2~3 個，MVP 不做 CSV 上傳）
- 資料表格（欄位名稱、型別、前幾列預覽）
- 各欄位基本統計（mean、std、min、max、count）
- 各欄位分布圖（histogram）

### 右欄（處理工具）
- 流程地圖：標示目前在哪個階段（資料描述 → 前處理 → 訓練/測試 → 結果解讀）
- 算法參數控制（k 值滑桿）
- 視覺化區塊（圖表即時更新）
- 基本操作 log（顯示做了什麼步驟，MVP 不需匯出）

---

## ML / 資料處理層面

### 資料描述（向下兼容中學生的出口）
- 各欄位 histogram
- mean、std、min、max、count

### 資料前處理（MVP 做一個：正規化）
- 正規化（Min-Max Scaling）
- **Before / After 並排視覺化**：同一欄位的分布圖前後對比，x 軸從原始範圍縮進 0~1，形狀不變
- 這是 DD 差異化最直接的展示，也是 DESIGN.md 五問的第一個驗證場

### 算法（MVP 做一個：k-NN）
- k-NN 分類
- k 值滑桿（即時更新決策邊界）
- 固定 80/20 train/test split（不讓學生調，降低認知負擔）

### 評估
- Accuracy
- 散點圖（標出預測對/錯）
- 決策邊界圖

## MVP 明確不做

| 項目 | 延後原因 |
|------|----------|
| CSV 上傳 | 增加資料 I/O 複雜度，內建資料集夠用 |
| 多算法 | 一個算法說清楚比多個算法說模糊有價值 |
| 標準化（Z-score） | 正規化先做，標準化視覺化邏輯類似，之後複用 |
| 操作 log 匯出 / MOSME 串接 | 教室系統是 MVP+1 |
| CSV 上傳 | 先用內建資料集驗證核心體驗 |
| k-NN 以外的算法 | 之後按教學故事逐一加入 |

## 內建資料集候選

需要同時滿足：統計層（中學生）和 ML 層（技職/大學生）都能說出有意義的故事。

| 資料集 | 優點 | 缺點 |
|--------|------|------|
| Iris | 經典、ml.js 有範例 | 對技職生較抽象（鳶尾花？）|
| 身高體重（台灣學生） | 生活感強，學生有感 | 需自己製作 |
| 成績資料（多科目 → 及格/不及格） | 學生最有共鳴 | 需自己製作 |

> 建議：MVP 用 Iris 先跑通技術，之後換成本土化資料集提升教學感。
